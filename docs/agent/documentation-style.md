# Documentation Style

`AGENTS.md`, `CLAUDE.md`, `docs/agent/*.md` 에이전트 참고 문서를 수정할 때만 사용한다.

## 에이전트 규칙 작성

기본 컨텍스트는 짧게 유지한다. 규칙은 해당 작업에서 에이전트가 이미 읽는 문서에 두고, 긴 배경 설명이나 예시는 선택 문서로 분리한다.

넓은 원칙보다 판단 기준이 있는 규칙을 우선한다. 가능하면 다음 순서로 쓴다.

- 의도: 왜 이 규칙이 필요한가
- 선호/회피: 기본적으로 어떤 선택을 할 것인가
- 적용 조건: 어떤 상황에서 적용하는가
- 예외: 언제 반대 선택이 가능한가
- 리뷰 질문: 작업 전 스스로 무엇을 물어볼 것인가

`clean`, `safe`, `proper`, `과하지 않게` 같은 모호한 표현은 판단 기준을 함께 주지 않는 한 쓰지 않는다.

## 문서 배치

- `AGENTS.md`와 `CLAUDE.md`는 라우터로 유지한다.
- 작업에 꼭 필요한 규칙은 가장 관련 있는 `docs/agent/*.md`에 둔다.
- 긴 예시는 선택 문서로 분리하고, 필요한 작업에서만 읽도록 라우팅한다.
- 짧은 타이브레이커 하나로 충분하면 새 always-read 문서를 만들지 않는다.

## 업데이트 체크리스트

- `AGENTS.md`와 `CLAUDE.md`가 라우터 역할만 유지하는가?
- 새 규칙이 실제 작업에서 읽는 문서에 배치되었는가?
- 원칙만 있고 적용 조건이나 예외가 빠져 있지 않은가?
- 오래된 코드 예시가 현재 API와 어긋나지 않는가?
- 긴 예시를 기본 문서에 넣어 컨텍스트 비용을 늘리지 않았는가?
- 사람만 읽을 배경 설명을 에이전트 기본 참고 문서에 넣지 않았는가?
- `.claude/settings*.json`이나 `docs/guide.md` 같은 도구 설정·운영 문서를 공통 작업 규칙으로 오해하게 만들지 않았는가?
